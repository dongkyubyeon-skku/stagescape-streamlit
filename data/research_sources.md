# StageScape Research Sources

## Research Purpose

This project was created to organize K-pop concert stage technologies from a production and visual direction perspective.

The goal was not to make a fan archive, but to build a structured reference archive for understanding how stage technologies are used in live performances.

---

## Main Research Sources

The data was collected and organized from publicly available sources.

### 1. Official Concert Videos

I mainly referred to official concert videos, official performance clips, concert trailers, and stage recordings.

These sources helped me observe the actual stage structure, screen usage, lighting design, and performance flow.

Examples:
- Official concert trailers
- Official live performance clips
- Tour highlight videos
- Award show performance videos
- Festival stage recordings

---

### 2. Official Artist and Company Channels

I also checked official YouTube channels and public content uploaded by entertainment companies.

Reference platforms included:
- HYBE Labels
- BANGTANTV
- SMTOWN
- JYP Entertainment
- YG Entertainment
- CJ ENM
- Mnet K-POP
- KCON Official
- Coachella official content

These channels were useful because they often show high-quality stage footage and official performance documentation.

---

### 3. Entertainment News and Articles

For basic information such as concert title, tour period, venue, and event scale, I referred to entertainment news articles and publicly available concert information.

These sources helped me organize:
- Concert year
- Venue
- Tour type
- Event scale
- Production context

---

### 4. Visual Observation

In many cases, exact technical specifications were not officially disclosed.

Because of this, I also used visual observation.

I watched performance videos and identified visible stage elements such as:
- Large LED screens
- Moving platforms
- Extended runways
- Projection-based visuals
- Drone performances
- Laser lighting
- Camera-based AR effects
- XR-style broadcast visuals

This process was used to assign technology tags to each concert case.

---

## Research Perspective

I analyzed the concerts from a stage production perspective.

Instead of focusing on:
- fan reactions
- song reviews
- idol popularity
- choreography only

I focused on:
- how the stage was structured
- what technologies were visible
- how visual effects supported the performance
- how venue scale affected stage design
- how the production created audience immersion

---

## Data Organization Method

Each concert case was organized using the following categories:

- Artist / Event
- Concert Title
- Venue
- Scale
- Audience
- Year
- Production Team
- Technology Tags
- Short Production Analysis
- Visual Reference Link

This structure allowed the archive to support two user flows:

1. Technology → Concert → Analysis
2. Concert → Technology → Analysis

---

## Limitations

Some stage technology information is not officially published.

Therefore, this archive should be understood as a prototype based on public research, visual analysis, and production-oriented interpretation.

The purpose is to create a structured archive model rather than a fully verified industrial database.
