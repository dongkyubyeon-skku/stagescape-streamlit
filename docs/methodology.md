# StageScape Methodology

## Project Objective

StageScape is a web-based archive project that organizes K-pop concert technologies and stage production information.

The main objective was to create a system where users can explore concerts not only by artist, but also by stage technology and production method.

---

## Research Question

The project started from the following questions:

- What kinds of stage technologies are commonly used in K-pop concerts?
- How do different concerts use similar technologies in different ways?
- How can concert production information be organized visually?
- How can users explore concerts from a production perspective?

---

## Data Collection Process

### Step 1. Selecting Concert Cases

I first selected representative K-pop concerts, world tours, award ceremonies, and festival performances.

The selection included:
- Large stadium concerts
- Arena tours
- Dome tours
- Award shows
- Global festivals
- Multi-artist events

This was done to compare different production scales.

---

### Step 2. Watching Visual References

I watched official concert videos, performance clips, stage recordings, trailers, and award show performances.

While watching, I focused on visible production elements such as:

- LED screen size and placement
- Stage layout
- Moving stage elements
- Runway structure
- Use of lighting and lasers
- Use of projections
- Drone performances
- Camera-based digital effects
- Broadcast-centered XR visuals

---

### Step 3. Creating Technology Tags

After observing the performances, I created a tag system to classify the technologies.

The main technology tags include:

- LED Wall
- XR
- AR Camera
- Projection Mapping
- Kinetic Stage
- Automated Lift
- Moving Runway
- Laser System
- Drone Show
- Virtual Production
- Pyrotechnics
- Hologram
- Follow Spot System
- Theatrical Set
- Graphic Screen Design

These tags were used to make the archive searchable and comparable.

---

### Step 4. Structuring the Dataset

Each concert was organized into a consistent data structure.

The data fields include:

- Artist / Event
- Concert Title
- Venue
- Scale
- Audience
- Year
- Production
- Technology Tags
- Production Analysis
- Visual Reference Link

Using a consistent structure allowed the website to generate filters, concert cards, and related technology analysis.

---

### Step 5. Designing User Flows

The website was designed around two main user flows.

#### Flow 1: Technology → Concert Analysis

Users select a technology tag first.

Then they can see concerts that use the selected technology.

This helps users compare how the same technology appears in different productions.

#### Flow 2: Concert → Technology Analysis

Users select a concert first.

Then they can see which technologies were used in that concert.

This helps users understand one concert as a complete production case.

---

## Visual Reference Strategy

At first, I tried to include actual concert images inside the website.

However, this caused several problems:

- image loading errors
- unstable external image links
- copyright concerns
- reduced website stability

Because of this, I changed the visual strategy.

Instead of directly embedding many images, I used:

- generated visual cards inside the website
- external visual reference links
- YouTube search links for actual stage footage

This made the website more stable and more realistic as a prototype archive.

---

## Analysis Criteria

The analysis was based on the following criteria:

### 1. Stage Technology

What visible technologies were used in the concert?

Examples:
LED Wall, XR, Drone Show, Projection Mapping, Moving Runway

### 2. Venue Scale

How large was the performance space?

Examples:
Arena, Dome, Stadium, Festival Stage, Award Ceremony

### 3. Production Function

How did the technology support the performance?

Examples:
- improving visibility
- creating immersion
- supporting narrative
- highlighting choreography
- connecting performers and audience

### 4. Visual Direction

What kind of visual identity did the concert create?

Examples:
- futuristic
- cinematic
- theatrical
- minimal
- festival-style
- broadcast-centered

---

## Reflection

Through this process, I learned that concert production can be analyzed as a combination of technology, design, space, and audience experience.

I also learned that data organization is important when building an archive.

The final website is not just a list of concerts, but a structured system that allows users to explore stage production from multiple directions.
